# 医疗数字化互动一级页面更新设计说明

## 目标

将现有一级页面更新为 GG 于 2026-07-17 确认的新版设计，并保留现有 React、Vite、腾讯云 COS 视频和 16:9 展示架构。

页面必须严格使用提供的设计素材，不增加返回按钮、播放控件、提示文字或其他可见网页控件。一级页面只有环形箭头持续运动，其余内容保持静止。

## 已确认的交互

- 左侧“机器人功能整合”图标直接播放对应全屏视频。
- 右侧“用户界面整合”图标直接播放对应全屏视频。
- 术前、术中、术后三张画面各分为左右两个独立点击区域。
- 点击每张画面的左侧或右侧彩色区域，直接播放该区域对应的视频。
- 点击正在播放的视频画面，返回一级页面。
- 下方矩形产品展示区域完全静止，不设置点击、动画或跳转。

## 一级页面视觉结构

一级页面继续使用固定 16:9 舞台，并按浏览器窗口等比例缩放。窗口比例不一致时允许舞台外出现黑边，不允许拉伸或裁切设计画面。

页面按以下图层从下到上组合：

1. `数字化互动2607161.jpg`：新版静态背景，包含标题、蓝紫渐变和底部产品展示矩形。
2. `1.png`、`2.png`、`3.png`：术前、术中、术后三组左右分区场景图。
3. `左边机器.png`、`右边手机.png`：左右两侧视频入口图标。
4. SVG 环形路径和移动箭头：覆盖在场景区域外围，持续匀速循环。
5. 八个完全透明的点击热区：覆盖在两个侧边图标和三组场景图的左右两半。

所有图层位置以 4000×2250 设计稿为视觉基准，运行时转换为相对百分比坐标。不得用近似 CSS 插画替换提供的图片。

## 环形箭头动画

- 使用 SVG 绘制与设计稿一致的圆角环形路径。
- 使用路径运动让箭头沿环形路径持续循环。
- 运动速度在整条路径上保持匀速，不在转角处暂停或加速。
- 动画无限循环，不响应鼠标悬停或点击。
- 箭头和路径不能遮挡标题、三组场景图、两侧图标或下方产品展示区域。
- 当系统启用 `prefers-reduced-motion: reduce` 时，保留路径和箭头，但停止持续运动，避免无障碍问题。

## 视频入口映射

一级页面共保留八个独立视频配置：

| 入口 | 页面配置标识 | 后续腾讯云文件用途 |
| --- | --- | --- |
| 机器人功能整合 | `video-robot-integration` | 机器人功能整合视频 |
| 用户界面整合 | `video-ui-integration` | 用户界面整合视频 |
| 技能培训 | `video-skills-training` | 术前技能培训视频 |

第一版让八个入口使用同一个可访问的简短 MP4 示例地址，确保播放交互可演示。每个入口仍保留独立配置项，后续只需替换对应的腾讯云 COS URL，不需要修改组件或导航逻辑。

## 视频播放行为

- 点击一级页面入口后，进入覆盖整个 16:9 舞台的视频页面。
- 视频自动播放，使用 `playsInline`，不显示浏览器原生控制条。
- 视频按完整屏幕显示，黑色作为不足区域的背景。
- 点击视频页面任意位置，立即返回一级页面。
- 视频播放结束后停留在最后一帧，不自动跳转。
- 视频加载失败时显示纯黑画面，点击仍可返回一级页面。
- 页面切换时停止并重置离开的视频，避免后台继续播放声音。

## 导航和状态

现有导航状态机继续负责页面切换，但一级页面不再先进入旧版术前、术中、术后静态二级场景。八个一级入口全部直接打开对应视频页，并将 `home` 压入历史栈。

从视频页返回时恢复同一个一级页面。快速重复点击期间只允许启动一次页面切换，防止多次压栈或叠加多个播放器。

旧版场景素材和路由暂不删除，降低回滚风险；它们不再由新版一级页面入口触发。

## 测试与验收

### 自动化测试

- 一级页面存在八个独立视频入口。
- 八个入口分别映射到正确的视频页面配置。
- 任一入口打开视频时，导航历史为 `home -> video-*`。
- 视频页面点击返回后恢复 `home`。
- 项目现有测试继续通过。
- Vite 生产构建成功。

### 浏览器验证

- 在 1920×1080 视口验证设计稿原始比例。
- 在一个非 16:9 视口验证无拉伸和无裁切。
- 检查标题、三张场景图、两侧图标、路径和底部矩形的位置。
- 检查环形箭头匀速连续循环。
- 依次点击八个入口，确认均能进入全屏示例视频。
- 在视频画面点击，确认均返回一级页面。
- 确认底部矩形没有点击行为。
- 检查控制台没有相关错误或警告。

## 回滚方式

旧版素材和旧版页面配置保留。若新版需要回滚，只需恢复 `src/data/stages.js`、`src/App.jsx`、`src/styles.css` 以及首页素材引用，无需迁移数据或修改腾讯云资源。
