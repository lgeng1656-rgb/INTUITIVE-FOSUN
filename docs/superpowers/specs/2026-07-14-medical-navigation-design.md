# 医疗互动跳转页面设计说明

## 目标

把现有医疗场景图片制作成一个可在本地浏览器运行的 1920×1080 触摸互动页面。画面必须严格沿用 GG 提供的设计图和按钮素材，不增加可见的网页控件、提示文字、边框或装饰。

第一版完成所有图片页面、透明点击热区、阶段切换、逐级返回和视频入口。四段正式视频稍后提供；在视频缺失期间，入口仍能进入统一的无文字黑色视频占位画面，并可点击返回上一页。

## 技术方案

- 保留现有 React 19 和 Vite 6 项目，不引入新的 UI 框架。
- 使用固定 1920×1080 舞台作为布局坐标系，通过 CSS 等比例缩放适配浏览器窗口。
- 原始设计图承担全部可见画面，React 只负责状态、透明热区、视频播放和返回逻辑。
- 热区位置使用相对于 1920×1080 舞台的百分比坐标，缩放后保持准确。
- 页面导航使用显式历史栈：进入场景或视频时压栈，点击空白画面时返回上一层。
- 底部阶段按钮拥有最高点击优先级；点击按钮直接打开对应阶段全场景，不触发页面返回。

## 页面流程

### 首页到阶段全场景

1. 点击首页术前画面，进入术前全场景。
2. 点击首页术中画面，进入术中全场景。
3. 点击首页术后画面，进入术后全场景。

### 术前流程

1. 术前全场景点击“技能培训”，进入技能培训场景。
2. 技能培训场景点击除阶段按钮外的画面，返回术前全场景。
3. 术前全场景点击“手术规划”，进入手术规划场景。
4. 手术规划场景点击“推想AI-4D手术规划系统”，进入推想“手术规划”视频。
5. 手术规划视频点击画面，返回手术规划场景。
6. 手术规划场景点击系统入口之外的画面，返回术前全场景。

### 术中流程

1. 术中全场景点击“远程教学”，进入远程教学场景。
2. 远程教学场景点击除阶段按钮外的画面，返回术中全场景。
3. 术中全场景点击“术中辅助”，进入睿术“术中导航”视频。
4. 术中导航视频点击画面，返回术中全场景。

### 术后流程

1. 术后全场景点击“手术复盘”，进入睿术“手术复盘”视频。
2. 手术复盘视频点击画面，返回术后全场景。
3. 术后全场景点击“质控管理”，进入睿术“数据分析”视频。
4. 数据分析视频点击画面，返回术后全场景。

### 阶段按钮

- 术前、术中、术后的全场景和图片详情场景均显示三个阶段按钮。
- 当前阶段使用对应点亮图，另外两个使用未点亮图。
- 点击阶段按钮时直接清空当前页面历史并打开目标阶段全场景。
- 视频页面不叠加阶段按钮，点击视频画面只返回上一页。

## 视频行为

需要的正式视频均为 MP4、1920×1080、16:9 横屏：

1. 推想“手术规划”视频。
2. 睿术“术中导航”视频。
3. 睿术“手术复盘”视频。
4. 睿术“数据分析”视频。

进入视频页面后自动播放，隐藏浏览器原生控制条。播放结束后停留在最后一帧，不自动跳转。点击视频画面返回上一页。正式视频文件缺失时显示纯黑画面，不显示虚构标题或提示文字。

## 交互优先级

同一次点击只允许执行一个动作，优先级从高到低为：

1. 阶段按钮切换。
2. 画面文字热区进入下一级。
3. 视频画面返回。
4. 普通图片页面空白区域返回。

所有热区在正常状态下完全透明，不显示调试边框。开发测试时可通过查询参数临时显示热区，但正式页面默认关闭。

## 视觉与适配要求

- 设计坐标为 1920×1080。
- 浏览器视口按 16:9 等比例缩放，不能改变图片宽高比。
- 宽高比不一致时允许页面外部出现黑边，不能裁切主画面。
- 禁止把按钮改成 CSS 绘制的近似样式。
- 禁止加入返回按钮、面包屑、标题栏、播放器控制条或悬停提示。
- 支持鼠标单击和触摸点击。

## 测试要求

- 首页三块场景图分别进入正确阶段。
- 一级首页下方说明文字不能触发跳转。
- 三个阶段按钮能互相切换，且亮暗状态正确。
- 技能培训、手术规划和远程教学图片场景能正确进入和返回。
- 四个视频入口映射到正确视频配置，缺少视频时仍可进入黑屏并返回。
- 热区点击不能被页面级返回事件抢占。
- 在 1920×1080 和一个非 16:9 浏览器窗口中，画面均保持比例且无裁切。
- 页面加载时无缺图、React 报错或浏览器控制台错误。

## 第一版范围外内容

- 不制作正式视频内容。
- 不增加后台管理、内容上传或网络接口。
- 不增加动画转场、音量设置、播放进度控制或自动返回首页。
- 不修改 GG 提供的原始图片内容。
